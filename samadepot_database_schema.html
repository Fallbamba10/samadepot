🎓 Flux étudiant — Déposer un travail

1
Connexion avec email universitaire
L'étudiant entre son email (ex: prenom.nom@ucad.edu.sn). Le système détecte automatiquement l'université via le domaine email.
Auth SupabaseEmail domain check
2
Voir les espaces de dépôt actifs
L'étudiant voit la liste de tous les espaces ouverts pour son département avec les deadlines en couleur (vert / orange / rouge).
RLS SupabaseDeadline visuelle
3
Sélectionner et uploader le fichier
Glisser-déposer ou sélection. Vérification format + taille côté client, puis upload vers Supabase Storage avec barre de progression.
Storage uploadValidation MIME
4
Confirmation + récépissé
Écran de succès avec le numéro de dépôt, l'horodatage exact et un lien pour télécharger le récépissé PDF.
PDF généréNotif WhatsApp
5
Suivi du statut en temps réel
Le dashboard étudiant affiche : Reçu → Lu → Validé / Retourné / Note attribuée. Notification à chaque changement.
RealtimeNotifications

👨‍🏫 Flux professeur — Évaluer un dépôt

1
Créer un espace de dépôt
Le prof définit : titre, type (devoir/examen/soutenance), deadline, formats acceptés, taille max. Un code d'accès unique est généré.
Code uniqueDeadline
2
Notification à chaque nouveau dépôt
Le prof reçoit un WhatsApp et/ou email dès qu'un étudiant dépose. Le tableau de bord se met à jour en temps réel.
WhatsAppEmailRealtime
3
Consulter et télécharger le fichier
Le prof ouvre la fiche du dépôt, voit les infos (nom étudiant, date, taille) et télécharge le fichier via URL signée temporaire (15 min).
URL signéeSécurisé
4
Donner son verdict
4 options : ✅ Valider · 📝 Valider + Note · 🔄 Retourner (corrections) · ❌ Rejeter. Un commentaire peut être ajouté.
ValiderRetournerNoter
5
Notification automatique à l'étudiant
L'étudiant est notifié par WhatsApp et email du verdict. Si retourné, il peut re-déposer une version corrigée.
WhatsApp étudiantRe-dépôt

1
Setup & Fondations
⏱ 3-4 jours
À faire en 1er
Initialiser le projet Next.js 14 + TypeScript + Tailwind CSS
Créer le projet Supabase + exécuter le schema.sql complet
Configurer l'authentification par email (Supabase Auth)
Mettre en place la détection d'université via le domaine email
Configurer le middleware de protection des routes par rôle
Déployer sur Vercel (staging)
2
Dépôt de fichiers (cœur du projet)
⏱ 1-2 semaines
MVP
Interface étudiant : liste des espaces de dépôt + deadlines
Zone d'upload avec glisser-déposer, validation format/taille
Upload sécurisé vers Supabase Storage + hash SHA-256 du fichier
Interface professeur : créer un espace de dépôt
Liste des dépôts reçus avec téléchargement sécurisé
Panneau d'évaluation : Valider / Retourner / Rejeter / Note
3
Notifications & Récépissé
⏱ 3-5 jours
MVP
Intégration WhatsApp Business API — notifications dépôt reçu
Notifications email via Resend — confirmation étudiant + prof
Génération du récépissé PDF horodaté avec hash du fichier
Notifications en temps réel dans l'interface (Supabase Realtime)
Notification étudiant à chaque changement de statut
4
Dashboard Admin Université
⏱ 1 semaine
Post-MVP
Vue globale : dépôts du jour, en attente, validés, statistiques
Gestion des comptes étudiants et professeurs
Gestion des départements et filières
Personnalisation du portail (logo, nom université)
Export des données en CSV/Excel
5
Multi-tenant + Super Admin
⏱ 1 semaine
Post-MVP
Ton dashboard : gérer toutes les universités inscrites
Onboarding d'une nouvelle université (formulaire + activation)
Statistiques nationales : dépôts totaux, universités actives
Gestion des plans et abonnements
Surveillance stockage global + alertes

Tests, sécurité & lancement pilote

Final
